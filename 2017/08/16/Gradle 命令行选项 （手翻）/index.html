Gradle 命令行选项 （手翻）

发表于 2017-08-16 | 分类于 杂项

英语水平渣, 凑合着看看吧…gradle工具命令行下的选项信息

USAGE: gradle [option...] [task...]

-?, -h, --help          帮助信息。
-a, --no-rebuild        不重建项目依赖关系。
-b, --build-file        指定build.gradle文件
--build-cache           使用Gradle构建缓存.Gradle将尝试复用以前版本的build结果. [incubating]
-c, --settings-file     指定setting.gradle文件
--configure-on-demand   仅配置必要的项目, Gradle将尝试缩减大型项目的配置时间 [incubating]
--console               指定输出类型,值为"plain"，"auto"（默认）或"rich"。
--continue              在一个Task失败后继续执行其他(不依赖失败Task的)Task.
-D, --system-prop       设置JVM的系统属性(例如-Dmyprop = myvalue).
-d, --debug             Debug模式(包括普通堆栈跟踪).
--daemon                使用Gradle守护程序运行构建.
--foreground            在前台启动Gradle守护进程. [incubating]
-g, --gradle-user-home  指定Gradle用户主目录.
--gui                   启动Gradle GUI.
-I, --init-script       指定初始化脚本.
-i, --info              设定日志等级为info.
--include-build         包含指定的构建[incubating]
-m, --dry-run           不运行任何task运行构建.
--max-workers           配置Gradle的最大并发数。[incubating]
--no-build-cache        禁用Gradle构建缓存. [incubating]
--no-daemon             禁用Gradle守护程序进行构建.
--no-scan               禁用创建构建扫描.(https://gradle.com/scans) [incubating]
--offline               在不访问网络资源的情况下执行构建.
-P, --project-prop      为构建脚本设置项目属性 (如-Pmyprop=myvalue).
-p, --project-dir       指定Gradle的起始目录, 默认为当前目录.
--parallel              并行构建项目,Gradle将尝使用最佳的线程数量. [incubating]
--profile               配置构建文件的执行时间,并在<build_dir>/reports/profile目录中生成日志.
--project-cache-dir     指定项目缓存目录。 在根项目目录中默认为.gradle。
-q, --quiet             只显示Error日志.
--recompile-scripts     强制进行重新构建.
--refresh-dependencies  刷新依赖状态.
--rerun-tasks           忽略先前缓存的Task结果.
-S, --full-stacktrace   输出所有异常的完整（非常详细）的堆栈跟踪.
-s, --stacktrace        打印出所有异常的堆栈跟踪.
--scan                  创建构建扫描。 如果构建扫描插件尚未应用，Gradle会发出警告。（https://gradle.com/scans）[incubating]
--status                显示运行中和最近停止的Gradle守护进程.
--stop                  停止正在运行的Gradle守护进程.
-t, --continuous        启用连续构建。 当Task文件输入更改时，Gradle不会退出, 并重新执行任务。 [incubating]
-u, --no-search-upward  不在父文件夹中搜索settings.gradle文件。
-v, --version           输出版本信息.
-w, --warn              设置日志等级为warn.
-x, --exclude-task      指定要从构建中排除的Task.
